## Task 4: Backfill `read_date` from git history

**Goal:** Compute the earliest-add git commit date for each booknote MD, write to frontmatter, regenerate JSON, commit.

- [ ] **Step 1: Compute read_date for all files via shell**

Run this shell script and capture output:

```bash
mkdir -p docs/superpowers/audits/scratch/
REPORT=docs/superpowers/audits/scratch/phase3-read-date-flagged.md
echo "# Phase 3 read_date computation results" > "$REPORT"
echo "" >> "$REPORT"
echo "Captured: 2026-05-14" >> "$REPORT"
echo "" >> "$REPORT"
echo "Files where --follow result differs from --all result, or where no date was found, are flagged." >> "$REPORT"
echo "" >> "$REPORT"
echo "| File | --follow date | --all earliest | Match? | Action |" >> "$REPORT"
echo "|------|---------------|----------------|--------|--------|" >> "$REPORT"

> /tmp/read-dates.txt

for f in docs/salt/*.md; do
  follow_date=$(git log --diff-filter=A --follow --format="%aI" -- "$f" 2>/dev/null | tail -1 | cut -c1-10)
  all_date=$(git log --all --diff-filter=A --format="%aI" -- "$f" 2>/dev/null | tail -1 | cut -c1-10)

  if [ -z "$follow_date" ]; then
    echo "$f|unknown|UNTRACKED" >> /tmp/read-dates.txt
    echo "| $f |  |  | n/a | **set to \"unknown\"** |" >> "$REPORT"
  elif [ -z "$all_date" ] || [ "$follow_date" = "$all_date" ]; then
    # --all returned nothing extra, or matches --follow — clean case
    echo "$f|$follow_date|MATCH" >> /tmp/read-dates.txt
  else
    # mismatch — flag
    echo "$f|$follow_date|MISMATCH:$all_date" >> /tmp/read-dates.txt
    echo "| $f | $follow_date | $all_date | NO | **REVIEW — use earlier of the two** |" >> "$REPORT"
  fi
done

echo "Total files processed: $(wc -l < /tmp/read-dates.txt)"
echo "Clean: $(grep -c '|MATCH$' /tmp/read-dates.txt)"
echo "Flagged: $(grep -c -v '|MATCH$' /tmp/read-dates.txt)"
```

Expected: ~106 files processed; most should be `MATCH`; a few might be `MISMATCH` (delete-readd cases) or `UNTRACKED` (any recently-added uncommitted booknote).
